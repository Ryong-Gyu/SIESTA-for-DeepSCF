# Custom SIESTA Extensions

This repository provides **custom SIESTA extensions** developed for the
[*DeepSCF approach*](https://doi.org/10.1038/s41524-024-01433-0), a machine-learning strategy for accelerating density functional theory (DFT) calculations by learning the mapping from the initial guess electron density $\rho_0$ to the self-consistent field (SCF) electron density $\rho$.

The original codebase is based on [*SIESTA 4.1.5*](https://departments.icmab.es/leem/siesta/), with the additional functionalities described below.

## Features

### (1) Single-shot Khon-Sham calculations

These extensions enable SIESTA to read an external charge-density file and use it as the input density for a non-SCF or single-shot calculation.

- `SCF.Read.Charge`: If `.true.`, SIESTA reads the input charge-density file `In.RHO`.
- `SCF.Read.Charge.Fix`: If `.true.`, the imported charge density is kept fixed during the SCF procedure.

These options are mainly intended for non-SCF calculations using a predicted `RHO` file generated by DeepSCF.

A typical setup in the `input/RUN.fdf` is:

```
# Non-SCF calculation with predicted rho
MaxSCFIterations      0           # Default: 50
SCFMustConverge       .false.
User.Basis            .true.
SCF.Read.Charge       .true.
SCF.Read.Charge.Fix   .true.
```

### (2) Generate grid-projected input features

These extensions also allow SIESTA to generate grid-projected physical features used as inputs for DeepSCF.

- `SaveIonicCharge`: If `.true.`, SIESTA saves `Chlocal`
- `SaveNewFeature`: If `.true.`, SIESTA saves `rhoatom`, `rhoover`, `grd`, `tau`, `tau_tf`, `tau_vw`, `alpha`, `elf`

A typical setup in the `input/RUN.fdf` is:

```
#(7) Save feature without electronic structure calculations
MaxSCFIterations      0           # Default: 50
SCFMustConverge       .false.
SaveIonicCharge           .true.  # rho_ionic
SaveNewFeature            .true.  # physical features
```


## Notes
All modifications are designed to integrate seamlessly with the standard SIESTA workflow. 
For compilation and general usage instructions, please refer to the individual module directories or the [SIESTA documentation](https://docs.siesta-project.org/projects/siesta/en/stable/).
